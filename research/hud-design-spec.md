# HUD Visual Design Spec — Beneath the Overlay

*Reference for prose formatting. Forge Codex Entity #75.*

---

## What the HUD IS NOT

- NOT System-built. The System did not create it and cannot access it.
- NOT a blue popup screen. No tooltips, no windows, no gaming UI.
- NOT a bionic implant. No hardware. No cybernetics.
- NOT removable. Cannot be turned off.

## What the HUD IS

The conduit's interface, rendered through Nate's engineering brain as a fighter-pilot overlay. Data integrated into vision — seen THROUGH, not AT.

MIXED GEOMETRY: Hard angles (triangles, pronged targeting shapes) combined with rounded elements (concentric arcs, curved status bars, compass-rose circles, radial tick marks). The System's UI is all rigid boxes and sharp edges. Nate's HUD has organic curves because its source is organic. Technical but alive.

Affected by the world: flickers near strong System constructs, drops 0.3 seconds during Spirit moments, sharpens in weak-overlay zones, shakes during impacts.

## Visual References

### Reference 01 — Peripheral Data Panels
Eye-level view with data panels arranged around periphery of vision, not blocking center. Bar charts, network node maps, circular reticles, data matrices. Dense but organized. Blue-green tones. This density = Phase 3-4 (Analytical/Architectural). Note mix of rectangular panels AND circular elements.

### Reference 02 — Concentric Targeting
Cleaner, less cluttered. Concentric circles with radial tick marks centered on focal point. Node-and-line network map extending outward. Warm amber tones. This = Phase 2 (Tactical). Note organic curves of concentric rings mixed with geometric node connections.

### Key Takeaway
Data FROM THE PERSPECTIVE OF LOOKING OUT THROUGH THE EYE. HUD is part of vision, not a screen. Data lives at edges and sharpens on focus. Center stays clear for reality.

## Phase Progression

### Narrative Arc: Beta Build → Catches Up → Surpasses

The System's blue panels arrive polished, complete, professional. Nate's HUD arrives like firmware v0.1 — because it IS. The conduit is booting for the first time in a modern human. The System has had millennia to refine its UI. Nate's HUD is building itself in real time from his engineering brain.

The blue panels don't evolve. They were delivered finished. Nate's HUD is alive — it grows because its source is alive.

| Phase | Nate's HUD | System Blue Panels |
|-------|-----------|-------------------|
| Ch3 | DOS prompt | Polished, complete |
| Ch5-6 | Flickery but growing | Static, same as day one |
| Ch8 | Functional, rough | Still the same |
| Ch11 | More data, more control | Everyone's stuck with what they got |
| Ch15+ | Custom engineering workstation | Consumer product |

### Phase 1A — BOOT (Ch3, First Boot)
- Monochrome green, faint, barely noticeable
- Health and stamina only — two thin CURVED bars at lower periphery (arcs, not straight lines)
- No numbers. Just length = percentage
- Compared to the System's crisp blue panels everyone else gets, Nate's HUD looks like a DOS prompt next to Windows
- Flint isn't fully online yet — HUD is running on its own
- "A faint green shimmer, two curved lines like parentheses made of light."

### Phase 1B — CRAFT OVERLAY (Ch4, Room Zero)
- Trigger: First SA use on weapon racks / crafting materials
- Faint wireframe overlay ghosts onto objects Nate focuses on — structural data, stress points, material quality
- Monochrome green, rough, flickery
- The engineer's HUD asserting itself. Everyone else got a weapon menu. Nate got X-ray vision for how things are built
- Still no targeting, no threat data. Diagnostic tool, not combat tool

### Phase 1C — THREAT AWARENESS (Ch5-6, First Blood / Spike)
- Trigger: Combat pressure. The HUD responds to adrenaline
- Not targeting — subtle peripheral shift. Green at the edge of vision brightens in direction of incoming threats. Instinctive, not precise
- SA overlay gets cleaner with repeated use during crafting sessions
- First ACCIDENTAL TOGGLE (Ch6): During a crafting session, Nate gets overwhelmed by wireframe data on multiple objects. He mentally pushes it away — it dims. Pulls it back — it returns. Surprised. Doesn't fully understand it yet

### Phase 1D — TARGETING (Ch7-8, Meat Grinder / Graduation)
- Trigger: High-stakes combat in the final rooms
- Basic focal reticle — when Nate aims (crossbow), concentric arcs tighten on his focal point. Still monochrome green. Not a full targeting system — the HUD confirming what he's already looking at
- Flint starts PUSHING DATA — creature classifications, mana estimates, warnings appear as brief text flashes at periphery in Flint's sardonic shorthand
- Toggle control becomes semi-intentional — Nate can dim/brighten elements on purpose, though it still takes concentration
- By Tutorial end: functional but still monochrome, still rougher than blue panels. But growing FASTER than anyone's blue screens

### Phase 2A — COLOR (Ch9-11, post-Tutorial)
- Trigger: First time outside the Tutorial. Real world, real stakes
- Color bleeds in: amber for threats first (most survival-critical data). Blue-green for environmental/terrain. Base stays green
- Flint gains deliberate control — can highlight, suppress, prioritize what Nate sees. Shorthand develops ("Flint, clean it up" = strip to essentials)
- Toggle is now intentional. Nate can run SA overlay, threat awareness, or minimal mode
- THE CROSSOVER MOMENT (Ch10-11): Nate's HUD now displays MORE useful real-time data than blue panels. The beta build caught up. Reader recognizes it before Nate does

### Phase 2B — TACTICAL (Ch11-15)
- Full color palette operational
- THE STRINGS: thin cold white threads from every System rune back to central point
- Divine rune patterns in weak-overlay zones — luminous gold, warm, organic, curved
- Concentric targeting circles with radial tick marks sharpen on focus (Reference 02 style)
- Node-and-line network maps for System construct connections
- Nate's HUD now surpasses the System panels. The System gave everyone a consumer product. Nate's conduit built him a custom engineering workstation
- "The world had color now. Not paint or screens — the color of threat assessment."

### Phase 3 — ANALYTICAL (Books 2-4)
- Dense peripheral data panels (Reference 01 style)
- Structural analysis overlay, crafting ID, System process depth
- Basic rune reading. Labyrinth architecture detail beneath overlay

### Phase 4 — ARCHITECTURAL (Books 4-7)
- Full dual-layer: System (cold white rigid) and divine (warm gold organic) simultaneously
- Divine notation fluency. Full Labyrinth architecture visible

## Prose Naturalization Curve

The HUD's prominence in prose FADES as the reader absorbs it:

| Chapters | HUD prose density | Reader experience |
|----------|------------------|-------------------|
| Ch3-4 | High — describe every new element, it's novel | "What IS this thing?" |
| Ch5-8 | Medium — new elements described, established ones get shorthand | "It's growing" |
| Ch9-12 | Medium-low — color/tactical transitions get attention, routine elements just exist | "I know how this works" |
| Ch13-22 | Low — mentioned when it matters, otherwise invisible | "It's just how Nate sees" |
| Book 2+ | Minimal — only flagged when something NEW or BROKEN happens | Furniture |

## Prose Formatting Rules

DO: Describe experientially.
- "The curved green arc at his periphery had dropped to half."
- "Amber flared at the edge of his left eye — something from the tree line."
- "Cold white threads ran from the rune on the blade, thin as spider silk."
- "Concentric rings tightened as he focused, tick marks resolving into distance."

DO NOT: Screen-readout formatting. No "Health: 50%". No boxed stat displays.

EXCEPTION: System classification in italics inline — PROCESSING, ERROR, EXCEEDS MEASUREMENT RANGE. The System's cold language breaking through.

Glitch moments: visual disruption — static, flicker, brief blindness, color shift.

Critical rule: HUD never talks. It displays. Flint interprets. HUD = visual. Flint = verbal.
