# True Names & False Labels

**Codex draft — pending forge codex ingestion (entity_type: `lore`).**
**Drafted 2026-05-09. Ratified by David same date.**

---

## One-liner

Pre-Sealing names carry identity-power; the System overlays *labels* on top of originals; aligned operation = un-mislabeled use.

---

## The mechanic

The Sealing introduced a labeling layer over a creation that was already named. Pre-Sealing, the Labyrinth-as-pipeline delivers an already-named architecture: every faculty, every Gift, every vocation, every bond carries an original name in a high-fantasy / angelic-language register — the speech of the original framework's making. Names in this register are not metadata; they are identity, function, and dominion fused.

(Cf. Tolkien's framing of elvish as analogous to angelic language; Le Guin's Old Speech; Genesis-Adam naming the animals as dominion-with-care; Isaiah 43:1; Revelation 2:17 white-stone-with-a-new-name.)

The System is, definitionally, a *labeler*. It reaches for things already named and writes its own names on top — class designations, skill nameplates, category tags, schema entries. The function is real; the name stored against it is a label, not the truth.

---

## Three name-tiers

1. **True names** (Labyrinth-delivered, pre-Sealing, unrebrandable when held in alignment): faculty identities (Nate's stockman's sense, his maker's eye, his pattern-reading), vocational callings, true bonds (the pack-bond), Spirit-direct Gifts (these bypass the Labyrinth and carry their original names from above).
2. **Labels** (System-issued, overlays): class designations (Vanguard, Warrior, Beast Master), bolded skill/ability nameplates (Fire Bolt, Field Repair), ranks, status flags, all bold-text System UI vocabulary.
3. **Mixed cases:** a character may have a true-name faculty under a System label (System sees only the label; the user *feels* the true thing underneath when aligned). A Conduit Gift may exist alongside a System-knockoff name (narrower, weaker, brand-mislabeled), but the original Gift name rules when accessed Spirit-direct.

---

## Mechanic consequences

- **Conduit overflow = un-mislabeled use.** The System's caps are calibrated to its labels, not the true thing underneath. Aligned operation reaches past the label and exceeds System-rated ceilings because the label was never the real measure.
- **Allegiance routing under a label** routes worship/credit through the label-issuer (the System / Morningstar). True-name use routes correctly to the Source.
- **Demonic name-extraction** (Mark 5: "What is your name?" "Legion") is a known sub-mechanic. Names taken compel; names refused or hidden cannot. By extension, the System's labeling of a person is a soft-coercive analog of the same move.

---

## Two-tier naming for Nate's faculties

Nate's *common name* for a faculty stays rancher-authentic and in-character (e.g., "stockman's sense" — cattle-rancher universal — NOT "shepherd's-eye" which is sheep-coded and out-of-vocabulary for a working cattleman).

The *Labyrinth-true name* (capital, codex-tier) carries the high-fantasy / Tolkien-Rohirric weight. Candidate term: **Heordsight** (Anglo-Saxon *heord* = herd, universal across all livestock). Other candidates: **the Wardenship**, **the Keeping**, **Stockwise**.

True names live in the codex, surface in prose only when earned.

## Reveal arc

- **Book 1 early (current):** no character has framework. Nate registers Layer 3 overrides as relabeling without naming the mechanic. Word: "the original," "the one already there," "the name beneath."
- **Book 1 mid:** Flint begins hedging — "what I'm reading looks more like a label than a name" — within the Flint design-intent throttle.
- **Book 1 late / Book 2:** the mechanic surfaces explicitly through experience — a true-name use produces an effect the System cannot account for.
- **Book 2+:** the term "true name" can be earned in prose.

## Editorial telltales

- Class names, skill names, **System UI bold-text** → label register.
- Faculty names, vocational identities, bonds → true-name register.
- Prose-register shift between the two tracks alignment state without explanation.
- The "**re-**" prefix in System action verbs ("re-shelved," "re-skinned," "re-categorized") and the spatial "**over**" preposition ("a label sliding into place *over* the original") are the grammatical signatures of false-naming.
- **Villain palette:** false-naming is the System's signature corruption. Villains *call things by the wrong name* and force-route allegiance through the mislabel.
- **Rancher-vocabulary check** — when picking a common-name for a Nate-faculty, confirm it isn't out-of-group for a cattleman (e.g., "shepherd" = sheep, not cattle). Universal stockman terms only.
